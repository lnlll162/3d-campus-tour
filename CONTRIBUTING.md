# Contributing to 3D校园云旅游系统

感谢你对本项目的关注与贡献！为保证协作顺畅，请遵循下面的流程与规范。

## 报告问题（Issues）
- 先搜索现有 Issue，避免重复提交
- 若是 Bug，请创建 issue 并填写必要信息（复现步骤、环境、日志）
- 若是 新功能，请说明场景与预期行为

## 开发流程
1. Fork 本仓库
2. 创建特性分支：`git checkout -b feature/your-feature`
3. 提交修改：`git commit -m 'feat: 描述你的改动'`
4. 推送分支：`git push origin feature/your-feature`
5. 创建 Pull Request，描述改动与测试步骤

## 代码规范
- 使用 ESLint（项目已配置）
- 使用语义化提交信息（feat/fix/docs/chore/...）
- 所有 PR 需通过 CI 检查（lint/test）

## 代码审查
- PR 会被指定评审者进行 Review
- 通过 Review 后才可合并到主分支

## 本地运行
```bash
npm install
npm run dev
```
